Congratulations!

You successfully created a solution consisting of 3 projects to help you testing web applications with Selenium and Specification by Example techniques using SpecFlow.

Complete the following steps to get started:

  1. Right-click on the solution
  2. Chose to Restore NuGet Packages
  3. Build the solution to see that it's all working

Then you are ready to start the real work: adding code to test your first web page.
As an example we will test the Google homepage.

  1. Expand the Pages project and right-click on the ModuleA folder
  2. Chose Add > New item > SpecByExample PageObject
  3. As a name you just enter Google and press Enter
  4. In the next wizard you enter the url: www.google.com and click Next
  5. For the next 3 steps you just accept the defaults by clicking Next
  6. On the last Wizard page you select both checkboxes to create a SpecFlow Steps and Features file.
  7. Click Finish to close the Wizard

A set of new files is now created in your project:

  • The Pages project contains a new file called GooglePage.cs containing the PageObject implementation for automating the Google homepage.
  • In the Specs project a similar folder ModuleA is created
  • Within that folder a specs file is created with GoogleFeature.feature with an example of a scenario (executable as a unit test)
  • And a file GoogleSteps.cs implementating the Given/When/Then steps to arrange, act and assert.

The proof of the pudding is to build the project, reopen the Test Explorer and run MyFirstTest.
This should open a web browser, go to Google, then click a random link and check the navigation actually occurred.

This extension can be found under Tools > Extensions and Updates and is named "Specification By Example Extension".
